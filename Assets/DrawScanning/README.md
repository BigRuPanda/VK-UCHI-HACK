# DrawScanning — Модуль распознавания рисунков для Unity WebGL

Распознавание рисунков в реальном времени для Unity WebGL и редактора.  
Ребёнок рисует что-то на листе бумаги, подносит к веб-камере и совмещает с полупрозрачным трафаретом на экране. Модуль определяет тёмные линии рисунка и оценивает, насколько точно он совпадает с эталонным трафаретом — всё на стороне клиента, без сервера.

---

## Архитектура (Stencil Matching)

Весь CV-пайплайн работает нативно на C# через Unity `WebCamTexture`. Это означает **полную работоспособность в редакторе Unity** для быстрого тестирования, а также в WebGL.

### Алгоритм

1. **Адаптивная бинаризация** — строится интегральное изображение для быстрого вычисления локального среднего в каждом блоке. Пиксель считается «нарисованным», если он значительно темнее своего окружения. Это обеспечивает устойчивость к неравномерному освещению.

2. **Flood fill для заполнения контуров** — стек-based flood fill от краёв изображения находит фон, инвертирует результат — получаются заполненные фигуры, а не только контуры.

3. **Сравнение с трафаретом** — подсчитывается процент «нарисованных» пикселей внутри трафарета (Coverage) и штраф за пиксели вне трафарета (Penalty). Итоговый счёт: `score = coverage - penalty × penaltyWeight`.

4. **Оптимизация производительности** — кадр с камеры уменьшается до небольшого `RenderTexture` (например, 128×128) перед обработкой. Это обеспечивает 60 FPS даже в WebGL на слабых устройствах.

## Быстрый старт (5 минут)

### 1. Подготовить эталонное изображение

- Создать или экспортировать **чёрный контур на белом фоне** (PNG) или **чёрный контур на прозрачном фоне** (PNG) целевой формы (например, контур моста).
- Импортировать в Unity. В **Texture Import Settings**:
  - `Texture Type` → **Default**
  - ✅ **Read/Write Enabled**

### 2. Добавить компонент в сцену

1. Создать пустой GameObject (например, `DrawScanningChecker`).
2. Добавить компонент `DrawScanningChecker`.
3. Перетащить эталонный PNG в поле **Reference Image**.
4. Создать `RawImage` для превью камеры и назначить в **Camera Preview UI**.
5. Создать ещё один `RawImage` для оверлея и назначить в **Overlay UI** (скрипт автоматически сделает его полупрозрачным).
6. (Опционально) Создать третий `RawImage` для отладки и назначить в **Processed Preview UI** — будет показывать то, что «видит» алгоритм.

### 3. Подключить события

| Событие | Когда срабатывает | Рекомендуемый обработчик |
|---|---|---|
| `OnSimilarityUpdate (float)` | Каждый обработанный кадр | Обновить полоску прогресса |
| `OnDrawingAccepted ()` | Однократно, когда счёт стабильно выше порога | Запустить магическую анимацию, перейти к следующей сцене |
| `OnCameraError ()` | Доступ к камере запрещён / недоступна | Показать UI «разрешите доступ к камере» |

### 4. Сборка для WebGL

- **File → Build Settings → WebGL**
- Раздавать билд через **HTTPS** (или `localhost`) — браузеры требуют защищённый контекст для доступа к камере.

## Справочник Inspector

### Настройки алгоритма

| Поле | По умолчанию | Описание |
|---|---|---|
| `processingResolution` | `128` | Разрешение обработки. Меньше = быстрее. 128 или 256 рекомендуется для WebGL. |
| `acceptThreshold` | `0.55` | Порог схожести (0–1) для срабатывания `OnDrawingAccepted`. Меньше = проще для детей. |
| `adaptiveBlockSize` | `45` | Размер блока для адаптивной бинаризации (нечётное число). Больше = устойчивее к неравномерному освещению. |
| `adaptiveC` | `7` | Константа, вычитаемая из локального среднего. Больше = меньше шума от текстуры бумаги. |
| `penaltyWeight` | `1.5` | Множитель штрафа за рисование вне трафарета. Предотвращает обман закрытием камеры. |

### Настройки камеры

| Поле | По умолчанию | Описание |
|---|---|---|
| `cameraWidth` | `640` | Запрашиваемая ширина камеры. |
| `cameraHeight` | `480` | Запрашиваемая высота камеры. |
| `cameraFPS` | `30` | Запрашиваемый FPS камеры. |
| `processingInterval` | `0.1` | Секунды между обработкой кадров (0.1 = 10 fps). Увеличить для экономии CPU. |

## Советы по калибровке

| Симптом | Решение |
|---|---|
| **Камера серая в редакторе** | Убедиться, что веб-камера подключена и не занята другим приложением (Zoom/Discord). |
| **Счёт не растёт** | Возможно, слишком тёмное освещение или рисунок недостаточно тёмный. Попробовать увеличить `adaptiveC` до 10–15. |
| **Счёт всегда высокий (ложное срабатывание)** | Фон слишком тёмный или `penaltyWeight` слишком мал. Попробовать увеличить `penaltyWeight`. |
| **Игра тормозит в WebGL** | Увеличить `processingInterval` до `0.2` (5 fps) или уменьшить `processingResolution` до `64`. |

## Публичный API

```csharp
// Запустить сканирование
checker.StartScanning();

// Остановить сканирование
checker.StopScanning();

// Сбросить флаг принятия (для повторного прохождения)
checker.ResetAcceptedState();

// Запросить разрешение камеры (вызывается из кнопки UI)
checker.RequestCameraPermission();

// Текущий счёт [0..1]
float score = checker.CurrentScore;

// Было ли рисование принято
bool accepted = checker.IsAccepted;
```
